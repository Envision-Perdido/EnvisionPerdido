# Play Perdido UI/UX Strategy & Homepage Wireframe
## Build-Ready Direction for a Best-in-Class Community Discovery Site

---

# 1. North Star

Play Perdido should not feel like a government calendar, a chamber directory, or a plugin page.

It should feel like:

> **A beautiful, fast, curated discovery engine for getting wonderfully lost in Perdido.**

The site should make visitors and locals feel:
- There is more to do here than they realized.
- Perdido has its own identity, not just overflow from Pensacola or Orange Beach.
- Staying local is easy, rewarding, and fun.
- Envision Perdido is actively helping the community thrive.

---

# 2. Core Product Promise

## Primary Promise

**Find what’s happening in Perdido, right now.**

## Emotional Promise

**Get lost in the good stuff.**

Perdido means “lost,” and that should become part of the brand logic. The site should invite people to get lost in:
- beach life
- live music
- food trucks
- local markets
- family activities
- nature
- local businesses
- community events

# 4. Strategic UX Positioning

The Chamber site starts with a tourism promise and collapses into a business directory.

Play Perdido must do the opposite:

1. Start with beauty and discovery.
2. Immediately show useful events.
3. Make filtering feel easy.
4. Keep users inside a curated local experience.
5. Make partner sites want to embed the system.

---

# 5. Site Architecture

## Primary Navigation

Keep the top navigation simple.

Recommended nav:

- Today
- This Weekend
- Live Music
- Food Trucks
- Family
- Submit Event

Secondary links:
- About
- For Partners
- Add This Calendar
- Envision Perdido

## Why This Works
The navigation maps to user intent, not organizational structure.

Users do not arrive thinking:
- “I need a directory.”
- “I need a chamber category.”

They arrive thinking:
- “What can we do tonight?”
- “What’s good this weekend?”
- “Where is live music?”
- “What can I do with kids?”
- “Where should we eat?”

# 6. Homepage Strategy

## Homepage Goal

The homepage should answer three questions in the first few seconds:

1. What is this?
2. What can I do right now?
3. Why should I stay in Perdido?

## Homepage Layout Overview

1. Hero / discovery search
2. Today + tonight quick picks
3. This weekend feature section
4. Category discovery tiles
5. Live music spotlight
6. Food truck / local flavor cross-link
7. Event feed with filters
8. Partner embed callout
9. Submit event / musician schedule callout
10. Envision Perdido mission footer

---

# 7. Homepage Wireframe

## Section 1: Hero

### Layout
Full-width beach or lifestyle image, with a subtle dark gradient overlay.

### Headline
**Get Lost in Perdido**

### Subheadline
**Find local events, live music, food trucks, family fun, and things to do today, tonight, and this weekend.**

### Primary CTA
**See What’s Happening**

### Secondary CTA
**Live Music Tonight**

### Optional Microcopy
**Built by Envision Perdido to help visitors and locals discover more right here at home.**

### UX Notes
- Avoid a giant hero that pushes events too far down.
- Hero should be beautiful but compact.
- On mobile, hero should fit in roughly one screen with first event teaser visible below.

---

## Section 2: Quick Discovery Bar

Immediately below hero.

### Layout
Horizontal pill buttons on desktop, swipeable row on mobile.

### Buttons
- Today
- Tonight
- This Weekend
- Live Music
- Family Friendly
- Free
- Food & Drink
- On the Water

### UX Notes
This creates instant usefulness. It should feel like a concierge, not a database.

---

## Section 3: Today in Perdido

### Purpose
Make the site useful immediately.

### Layout
EventON-inspired compact list or small cards.

### Recommended Display
Use a clean EventON list-style layout, but upgraded with:
- larger date blocks
- stronger venue line
- category chips
- optional thumbnail
- clear CTA

### Event Card Elements
- date/time block
- event title
- venue/location
- short AI-enhanced teaser
- category badge
- “View Details” or expand affordance

### Interaction
Inline expansion is acceptable here. EventON’s accordion model works well for fast browsing.

### UX Notes
This section should prioritize density and speed over visual drama.

---

## Section 4: Featured This Weekend

### Purpose
Add editorial curation and visual excitement.

### Layout
Mixed-aspect card mosaic.

Suggested desktop layout:
- One large feature card, 2:1 horizontal
- Two medium cards, 4:3
- Two smaller cards, 1:1 or 3:4

Suggested mobile layout:
- Stacked cards
- First featured event large
- Remaining cards in compact vertical feed

### Why Mixed Aspect Ratios Matter
A uniform grid feels like a directory. A mixed mosaic feels curated and alive.

### Card Treatment
- full-bleed image
- gradient overlay
- title in bold display type
- time + venue
- small tag: “Top Pick,” “Family,” “Live Music,” etc.

### UX Notes
This is where Play Perdido should visually outclass the Chamber.

---

## Section 5: Choose Your Perdido

# 8. Event Card System

## Three Card Types

### 1. Compact List Card
Best for:
- Today
- Tonight
- dense event feeds

Structure:
- date/time block
- title
- venue
- category chip
- short teaser
- expand/detail action

Use EventON’s accordion behavior here.

---

### 2. Visual Feature Card
Best for:
- top picks
- weekend highlights
- homepage mosaic

Structure:
- image
- gradient overlay
- title
- time
- venue
- featured badge

Aspect ratios:
- 2:1 hero card
- 4:3 standard card
- 1:1 square card
- 3:4 portrait card

Use mixed ratios intentionally.

---

### 3. Detail / Conversion Card
Best for:
- event detail pages
- ticketed events
- musician shows

Structure:
- large image
- full description
- map
- organizer
- add to calendar
- share
- ticket / RSVP / website CTA

# 9. EventON Integration Strategy

## Use EventON For
- event storage
- recurring event logic
- event metadata
- category system
- filtering
- shortcode-driven views
- expandable event details

## Customize Around EventON
- homepage layout
- visual card styling
- featured event sections
- category landing pages
- partner embed styling

## Do Not Overbuild Initially
EventON already has usable interaction patterns:
- expandable lists
- tiles
- category filters
- calendar/list views

The goal is not to replace EventON. The goal is to wrap it in a better discovery experience.

- coastal
- modern
- warm
- polished
- local

## Color Direction
Use Perdido-inspired colors:
- Gulf blue
- deep water blue
- seafoam
- warm sand
- sunset coral
- white

Use a strong display type sparingly for headlines, but avoid the overused tourist-board look.

- readable card typography

## Imagery
Prioritize:
- beach
- sunset

- markets

# 13. Content Strategy

## Event Titles
Clean titles, not raw scraped titles.

Bad:
“Blue Angels Practice . Skip the Traffic . Skip the Crowds . Watch from the Water”

Better:
“Blue Angels Practice Watch Party on the Water”

## Event Descriptions
Use AI-assisted cleanup to create:
- short teaser
- useful detail
- local context

## Category Labels
Use user-friendly categories:
- Live Music
- Family Friendly
- Food & Drink
- Free
- On the Water
- Markets
- Community
- Arts & Culture

Avoid internal categories:
- committee
- networking
- member event
- chamber event

---

# 14. Curation Rules

An event should be included if it is:
- in Perdido or directly serving Perdido
- open to the public
- community-facing
- useful to locals or visitors

An event should be excluded or downgraded if it is:
- outside the Perdido area
- primarily an advertisement
- private/internal
- spammy
- not useful to visitors or locals
- primarily designed to send people out of the local economy

---

# 15. Differentiation From Chamber Site

## Chamber Site Problem
The Chamber site begins as a tourism experience and collapses into a business directory.

## Play Perdido Response
Play Perdido must remain a discovery experience all the way through.

| Chamber Pattern | Play Perdido Alternative |
|---|---|
| Generic listings | Curated event cards |
| Member-first | User-first |
| Out-of-area leakage | Perdido-first |
| Directory UX | Discovery UX |
| No prioritization | Top picks and quick filters |
| Static pages | Living event feed |

# 19. Prompt Guidance for Future GPT / Codex Sessions

When using this document as input for another GPT or developer session, emphasize:

- This is not a generic calendar site.
- This is a curated local discovery engine.
- EventON should be used as the event backend and interaction foundation.
- The frontend should remain visually polished, mobile-first, and discovery-oriented.
- Do not reproduce Chamber directory patterns.
- Prioritize Perdido-first economics and geographic integrity.
- Use mixed card aspect ratios for visual energy.
- Maintain low admin burden through automation and moderation.
- Build reusable components for partner embeds and category landing pages.
