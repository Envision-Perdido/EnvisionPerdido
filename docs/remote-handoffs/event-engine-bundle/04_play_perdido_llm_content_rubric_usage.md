# Play Perdido LLM Content Rubric Usage
## Content Generation Guardrails for Codex and LLM Workers

---

# Purpose

This document explains how to use the Play Perdido content rubric during site build work.

Primary source:

- [play_perdido_llm_content_guidelines_rubric.md](/home/steve/Desktop/Documents/Envision Perdido/build/playperdido/tools/play_perdido_llm_content_guidelines_rubric.md)

Use this whenever content is being generated for:

- homepage sections
- event teasers
- category intros
- CTA copy
- empty states
- partner blurbs
- submission copy
- card microcopy
- short and medium guide content

Exception:

- do not use it as the primary guide for full adventure articles when a separate adventure-specific content process is required

---

# How It Should Be Used

Treat the rubric as a working tool, not optional reference material.

When generating content, Codex should check that the copy is:

- local
- useful
- clear
- discovery-oriented
- specific to Perdido
- aligned with `Get Lost in Perdido`

Codex should reject or revise copy that feels:

- generic
- chamber-like
- overhyped
- corporate
- filler-heavy
- disconnected from practical user decisions

---

# Operational Rule

For remaining implementation slices, generated content should follow this workflow:

1. identify the content type
2. use the rubric’s length band and voice rules
3. draft copy
4. self-review against the rubric
5. only then place the copy into templates, cards, sections, or UI states

This should apply to both:

- page-level copy
- UI microcopy

---

# Where This Applies In The Build

This rubric should actively guide work in:

- homepage refinement
- event branch pages
- music branch pages
- food / local flavor pages
- submit flows
- partner/support copy
- empty-state treatments

It should also guide:

- placeholder content used during development
- migration cleanup when imported content needs rewriting into the Play Perdido voice
